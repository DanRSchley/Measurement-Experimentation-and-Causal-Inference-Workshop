16 Part 1: What Do We Mean by “Causal Effect”?

16.1 Causal Inference is a Deductive Process

ImportantWhat this exercise assumes

The 64 structures shown below represent the complete space of possible causal relationships among three variables under ideal conditions: (1) proper measurement — X, M, and Y are each measured without discriminant validity failures or systematic error (Module 1), and (2) no omitted variables — these three variables are the only players in the system (Module 2). In real research, measurement error and hidden confounders expand this space considerably.

Causal inference is not induction — you cannot prove a causal structure from data. It is deduction by elimination: you bring every tool in your methodological toolkit to bear and see which structures survive. The exercise below makes this concrete.

The grid shows all 64 structurally distinct relationships that can exist among three variables X, M, and Y — arranged as a triangle with X at the lower-left, Y at the lower-right, and M at the top. Each cell shows one possible causal structure. Arrows between any two variables can point forward (A → B), backward (A ← B), bidirectionally (A ↔︎ B, representing correlated unmeasured causes or feedback), or not exist at all.

Click the buttons below to apply research findings. Each finding rules out the structures inconsistent with it — those cells will be crossed out in red. After exhausting your toolkit, notice how many structures remain. That irreducible remainder is why causal inference is hard.

// ── Generate all 64 causal structures ─────────────────────────────────────────
// Arrow code per pair: 0=none, 1=A→B (forward), 2=A←B (backward), 3=A↔B (bidir)
// Three pairs: xm (X↔M), my (M↔Y), xy (X↔Y)
dag_states = {
  const arr = [];
  for (let xm = 0; xm < 4; xm++)
    for (let my = 0; my < 4; my++)
      for (let xy = 0; xy < 4; xy++)
        arr.push({ xm, my, xy });
  return arr;
}

// ── Consistency logic ──────────────────────────────────────────────────────────
// Returns true if state s is consistent with all active tool findings.
//
// Arrow code per pair: 0=none, 1=A→B (forward), 2=A←B (backward), 3=A↔B (bidir)
// Pairs: xm (X,M), my (M,Y), xy (X,Y)
//
// Tool logic (each button represents a POSITIVE finding that rules out
// structures that cannot produce that result):
//
//   regDX — regression Y~X+M: X IS significant  → my≠0; xy≠0
//            (overridden if medD0 also active — mediation supersedes regression)
//   regD0 — regression Y~X+M: X NOT significant → my≠0; xy=0
//            (overridden if medDX also active — mediation supersedes regression)
//   expX  — X experimentally manipulated         → X exogenous: xm≠2,3; xy≠2,3
//   medDX — mediation, partial (direct ≠ 0)      → xm≠0; my≠0; xy≠0 (always enforced)
//   medD0 — mediation, pure   (direct = 0)       → xm≠0; my≠0; xy=0 (always enforced)
//
// Mediation supersedes regression on the xy constraint: if medD0 is active,
// regDX's xy≠0 requirement is waived; if medDX is active, regD0's xy=0 is waived.
function dagConsistent(s, active) {
  const { xm, my, xy } = s;
  const any = v => v > 0;

  // X exogenous when manipulated: no arrows into X
  if (active.includes("expX") && (xm === 2 || xm === 3 || xy === 2 || xy === 3)) return false;

  for (const t of active) {
    if (t === "regDX") {                                      // X→Y direct effect found
      if (!any(my)) return false;
      if (!active.includes("medD0") && !any(xy)) return false; // medD0 supersedes
    }
    if (t === "regD0") {                                      // no X→Y direct effect
      if (!any(my)) return false;
      if (!active.includes("medDX") && xy !== 0) return false; // medDX supersedes
    }
    if (t === "medDX" && !(any(xm) && any(my) && any(xy))) return false; // partial mediation
    if (t === "medD0" && !(any(xm) && any(my) && xy === 0)) return false; // pure mediation
  }
  return true;
}

// ── Main DAG grid ──────────────────────────────────────────────────────────────
dagGrid = {
  // Cell geometry
  const W = 86, H = 70, GAP = 4;
  // Node positions within each cell (origin = cell top-left)
  const NX = [15, 58], NY = [71, 58], NM = [43, 12];
  const NR = 9, APAD = 10;

  // Layout metrics
  const LPAD = 58, SGAP = 16;
  const SHEAD = 26, CHEAD = 16;
  const secW = 4 * (W + GAP) - GAP;   // 352
  const secH = 4 * (H + GAP) - GAP;   // 292
  const totalW = LPAD + 2 * secW + SGAP + 4;
  const totalH = 2 * (SHEAD + CHEAD + secH) + SGAP + 6;

  const svg = d3.create("svg")
    .attr("width", totalW).attr("height", totalH)
    .style("display", "block").style("overflow", "visible")
    .style("font-family", "system-ui, -apple-system, sans-serif");

  // Arrowhead markers — one normal, one faded
  svg.append("defs").html(`
    <marker id="dah"  viewBox="0 0 8 8" refX="6" refY="4"
            markerWidth="5" markerHeight="5" orient="auto">
      <path d="M0,1 L0,7 L8,4 z" fill="#374151"/>
    </marker>
    <marker id="dahf" viewBox="0 0 8 8" refX="6" refY="4"
            markerWidth="5" markerHeight="5" orient="auto">
      <path d="M0,1 L0,7 L8,4 z" fill="#d1d5db"/>
    </marker>
  `);

  // Draw a single directed arrow from (ax,ay) to (bx,by) with perpendicular offset
  function putArrow(g, ax, ay, bx, by, off, faded) {
    const dx = bx - ax, dy = by - ay;
    const len = Math.sqrt(dx * dx + dy * dy) || 1;
    const ux = dx / len, uy = dy / len;
    const ox = -uy * off, oy = ux * off;
    g.append("line")
      .attr("x1", ax + ux * APAD + ox).attr("y1", ay + uy * APAD + oy)
      .attr("x2", bx - ux * APAD + ox).attr("y2", by - uy * APAD + oy)
      .attr("stroke", faded ? "#d1d5db" : "#374151")
      .attr("stroke-width", faded ? 1.1 : 1.4)
      .attr("marker-end", `url(#${faded ? "dahf" : "dah"})`);
  }

  // Draw one cell at position (cx, cy) in group g
  function drawCell(g, s, ok, cx, cy) {
    const gc = g.append("g").attr("transform", `translate(${cx},${cy})`);
    const f = !ok;

    // Cell background
    gc.append("rect")
      .attr("width", W).attr("height", H).attr("rx", 4)
      .attr("fill",   f ? "rgba(220,38,38,0.07)" : "#ffffff")
      .attr("stroke", f ? "rgba(220,38,38,0.28)" : "#e2e8f0")
      .attr("stroke-width", 0.8);

    // XM arrows
    if (s.xm === 1) putArrow(gc, NX[0],NX[1], NM[0],NM[1], 0, f);
    if (s.xm === 2) putArrow(gc, NM[0],NM[1], NX[0],NX[1], 0, f);
    if (s.xm === 3) {
      putArrow(gc, NX[0],NX[1], NM[0],NM[1], 3.5, f);
      putArrow(gc, NM[0],NM[1], NX[0],NX[1], 3.5, f);
    }
    // MY arrows
    if (s.my === 1) putArrow(gc, NM[0],NM[1], NY[0],NY[1], 0, f);
    if (s.my === 2) putArrow(gc, NY[0],NY[1], NM[0],NM[1], 0, f);
    if (s.my === 3) {
      putArrow(gc, NM[0],NM[1], NY[0],NY[1], 3.5, f);
      putArrow(gc, NY[0],NY[1], NM[0],NM[1], 3.5, f);
    }
    // XY arrows
    if (s.xy === 1) putArrow(gc, NX[0],NX[1], NY[0],NY[1], 0, f);
    if (s.xy === 2) putArrow(gc, NY[0],NY[1], NX[0],NX[1], 0, f);
    if (s.xy === 3) {
      putArrow(gc, NX[0],NX[1], NY[0],NY[1], 4, f);
      putArrow(gc, NY[0],NY[1], NX[0],NX[1], 4, f);
    }

    // Node circles + labels
    const ns = f ? "#d1d5db" : "#374151";
    const nf = f ? "#f9fafb" : "#ffffff";
    const tf = f ? "#9ca3af" : "#111827";
    [[NX,"X"],[NY,"Y"],[NM,"M"]].forEach(([pos, lbl]) => {
      gc.append("circle")
        .attr("cx", pos[0]).attr("cy", pos[1]).attr("r", NR)
        .attr("fill", nf).attr("stroke", ns).attr("stroke-width", 1.5);
      gc.append("text")
        .attr("x", pos[0]).attr("y", pos[1] + 4)
        .attr("text-anchor", "middle").attr("font-size", 10).attr("font-weight", "700")
        .attr("fill", tf).text(lbl);
    });

    // Red X cross for ruled-out structures
    if (f) {
      gc.append("line")
        .attr("x1", 5).attr("y1", 5).attr("x2", W-5).attr("y2", H-5)
        .attr("stroke", "rgba(220,38,38,0.28)").attr("stroke-width", 1.5);
      gc.append("line")
        .attr("x1", W-5).attr("y1", 5).attr("x2", 5).attr("y2", H-5)
        .attr("stroke", "rgba(220,38,38,0.28)").attr("stroke-width", 1.5);
    }
  }

  // Section labels and colours
  const XY_TITLES  = ["X — Y  (no direct link)", "X → Y", "X ← Y", "X ↔ Y"];
  const XY_COLORS  = ["#334155", "#1d4ed8", "#7c3aed", "#0e7490"];
  const XM_LABELS  = ["X—M: —", "X→M", "M→X", "X↔M"];
  const MY_LABELS  = ["—", "M→Y", "Y→M", "M↔Y"];

  // Render 4 sections in a 2×2 layout; each section = one XY relationship type
  [0, 1, 2, 3].forEach((xyVal, si) => {
    const sCol = si % 2;
    const sRow = Math.floor(si / 2);
    const ox = LPAD + sCol * (secW + SGAP);
    const oy = sRow * (SHEAD + CHEAD + secH + SGAP);

    // Section header bar
    svg.append("rect")
      .attr("x", ox).attr("y", oy).attr("width", secW).attr("height", SHEAD - 2)
      .attr("rx", 4).attr("fill", XY_COLORS[xyVal]);
    svg.append("text")
      .attr("x", ox + secW / 2).attr("y", oy + 17)
      .attr("text-anchor", "middle").attr("fill", "white")
      .attr("font-size", 12).attr("font-weight", "700")
      .text(XY_TITLES[xyVal]);

    // Column headers (MY relationship)
    for (let my = 0; my < 4; my++) {
      svg.append("text")
        .attr("x", ox + my * (W + GAP) + W / 2).attr("y", oy + SHEAD + CHEAD - 3)
        .attr("text-anchor", "middle").attr("font-size", 8.5).attr("fill", "#64748b")
        .text(MY_LABELS[my]);
    }

    // Row labels (XM relationship) + cells
    for (let xm = 0; xm < 4; xm++) {
      const cy = oy + SHEAD + CHEAD + xm * (H + GAP);
      // Row label on the left edge of each section column
      svg.append("text")
        .attr("x", ox - 5).attr("y", cy + H / 2 + 4)
        .attr("text-anchor", "end").attr("font-size", 8.5).attr("fill", "#64748b")
        .text(XM_LABELS[xm]);
      for (let my = 0; my < 4; my++) {
        const cx = ox + my * (W + GAP);
        drawCell(svg, { xm, my, xy: xyVal }, dagConsistent({ xm, my, xy: xyVal }, activeTools), cx, cy);
      }
    }
  });

  // Axis label: rows = XM relationship
  svg.append("text")
    .attr("transform", `translate(8, ${totalH / 2}) rotate(-90)`)
    .attr("text-anchor", "middle").attr("font-size", 10).attr("fill", "#94a3b8")
    .text("← XM relationship (rows within each section) →");

  return svg.node();
}

16.1.1 What if There Is an Unobserved Confounder?

The exercise above assumed no hidden variables. Now introduce U — an unobserved variable that may have arrows to X, M, and/or Y but is never measured. No research tool can constrain U directly.

Each of the three U-pairs (U–X, U–M, U–Y) can be absent, forward, backward, or bidirectional — 4³ = 64 U configurations for every one of the 64 X–M–Y structures, giving 4,096 total states. Rather than draw all 4,096 cells, the grid below preserves the familiar 64 X–M–Y layout. Each cell’s blue shading encodes how many of the 64 U configurations remain consistent with your findings: deeper blue = U is highly ambiguous for that structure; pale = almost no U configuration survives; crossed out = the X–M–Y structure is ruled out for every possible U configuration. The small dashed circle labelled U in each cell is a reminder that the confounder is always present but unobserved.

Notice in particular: pressing Manipulate X eliminates U→X (since X is now exogenous), but U’s connections to M and Y remain entirely free — experimental control of X does not eliminate unmeasured confounding of the M–Y path.

Code 
// ── Consistency logic for confounded model ─────────────────────────────────
// Six pairs: xm, my, xy (same as before) plus ux (U,X), um (U,M), uy (U,Y)
// Arrow codes: 0=none, 1=A→B, 2=A←B, 3=A↔B
//
//   regDX2 — regression: X IS significant    → my≠0; xy≠0 (overridden by medD02)
//   regD02 — regression: X NOT significant   → my≠0; xy=0 (overridden by medDX2)
//   expX2  — manipulate X                    → X exogenous: xm≠2,3; xy≠2,3; ux≠2,3
//   medDX2 — mediation, partial (direct≠0)   → xm≠0; my≠0; xy≠0 (always enforced)
//   medD02 — mediation, pure   (direct=0)    → xm≠0; my≠0; xy=0 (always enforced)
// Mediation supersedes regression on the xy constraint (same rule as sim 1).
dagConsistent2 = (s, active) => {
  const { xm, my, xy, ux, um, uy } = s;
  const any = v => v > 0;
  if (active.includes("expX2") && (xm===2||xm===3||xy===2||xy===3||ux===2||ux===3)) return false;
  for (const t of active) {
    if (t === "regDX2") {
      if (!any(my)) return false;
      if (!active.includes("medD02") && !any(xy)) return false;
    }
    if (t === "regD02") {
      if (!any(my)) return false;
      if (!active.includes("medDX2") && xy !== 0) return false;
    }
    if (t === "medDX2" && !(any(xm) && any(my) && any(xy))) return false;
    if (t === "medD02" && !(any(xm) && any(my) && xy === 0)) return false;
  }
  return true;
}

// Count how many of the 64 U configurations (ux, um, uy) are consistent
// for a fixed (xm, my, xy) state given the active button set.
countUConfigs = (xmyState, active) => {
  let n = 0;
  for (let ux = 0; ux < 4; ux++)
    for (let um = 0; um < 4; um++)
      for (let uy = 0; uy < 4; uy++)
        if (dagConsistent2({...xmyState, ux, um, uy}, active)) n++;

16.2 The Fundamental Problem of Causal Inference

Every participant in the AlterEco eco-label study had two potential outcomes:

  • \(Y_i(0)\): their WTP if they did not see the eco-label
  • \(Y_i(1)\): their WTP if they did see the eco-label

Their individual treatment effect is \(\tau_i = Y_i(1) - Y_i(0)\).

The problem? We can only observe one of these for any individual. This is called the Fundamental Problem of Causal Inference (Holland, 1986). Everything that follows is a strategy for getting around it.

16.4 Average Treatment Effect (ATE)

\[\text{ATE} = E[\tau_i] = E[Y_i(1) - Y_i(0)]\]

In a randomised experiment, random assignment makes the treated and control groups exchangeable, so a simple difference in group means is unbiased for the ATE.

Code

WarningConnection to Module 2

Module 2’s permutation test estimated the ITT — comparing all participants assigned to the eco-label condition against all assigned to control, regardless of whether they actually processed the label.
